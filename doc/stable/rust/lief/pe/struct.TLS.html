Pointer to an array of TLS callback functions.

The array is null-terminated, so if there is no callback function this field points to 4 bytes set to zero.

See TLS::callbacks

pub fn characteristics(&self) -> u32

The four bits [23:20] describe alignment info.

Possible values are those defined as IMAGE_SCN_ALIGN_*, which are also used to describe alignment of section in object files.

The other 28 bits are reserved for future use.

The initial content used to initialize TLS data.

pub fn addressof_raw_data(&self) -> (u64, u64)

Range of addresses where the TLS::data_template is located.

The template is a block of data that is used to initialize TLS data. The system copies all of this data each time a thread is created, so it must not be corrupted.

These addresses are not RVA. It is addresses for which there should be a rebase relocation in the .reloc section.
